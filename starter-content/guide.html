KapiLabs Astro Framework — Setup Guide

Everything you need to run this site on localhost and manage content in Sanity Studio.

Local URLs

After starting the dev servers, use these addresses:

Astro Website http://localhost:4321

Run: pnpm dev from project root.

Sanity Studio (CMS) http://localhost:3333

Run: npx sanity dev from studio/ folder.

Fresh Setup Steps

1

Create a free Sanity project

Go to sanity.io/manage → New Project. Copy your Project ID.

2

Set up .env file

Copy .env.example to .env in the project root. Fill in:

PUBLIC_SANITY_PROJECT_ID=your-project-id
PUBLIC_SANITY_DATASET=production
SANITY_TOKEN=your-write-token
SITE_URL=http://localhost:4321

Get a write token: sanity.io/manage → your project → API → Tokens → Add API Token (Editor role).

Also copy studio/.env.example to studio/.env and fill in SANITY_STUDIO_PROJECT_ID.

3

Install dependencies

pnpm install
cd studio && pnpm install
4

Deploy schema to Sanity

cd studio && npx sanity schema deploy

This registers all schema types so Studio knows how to render them.

5

Seed starter content

pnpm seed

Creates pages, posts, team members, testimonials, forms, and menus in Sanity.

Test first without writing: node scripts/seed-sanity.mjs --dry-run

6

Start dev servers

Terminal 1: pnpm dev → http://localhost:4321

Terminal 2: cd studio && npx sanity dev → http://localhost:3333

Project Structure

Two-Layer Source Split

src/core/ — framework. Never edit directly.

src/theme/ — your customizations. Shadow any core file here.

To override: copy file from src/core/ to same path in src/theme/ and edit there.

Troubleshooting

Page shows blank / no content

Check PUBLIC_SANITY_PROJECT_ID in .env. Run pnpm seed to add starter content.

Studio shows no schema types

Run npx sanity schema deploy from the studio/ folder.

Seed fails with SANITY_TOKEN missing

Create an Editor token at sanity.io/manage and add it to .env.

pnpm: command not found

Install pnpm: npm install -g pnpm
